# 项目背景
中国拥有上下五千年悠久的历史，也拥有大量的历史爱好者、研究者。地理是历史故事发生的舞台，是研习历史必不可少的一个要素。然而，历史著作上的原始地理信息往往晦涩繁杂，特别是古今地名差异、地点具体方位等内容，往往难以通过文字注释实时呈现。“史图记”项目启动的初衷，就是希望借助最新数字技术，可视化历史地理信息，提升历史爱好者、研究者，阅读相关著作的体验。
# 特色功能
“史图记”项目以《三国志》为起点，借助数字技术，以地图为主要传播媒介，实现了以下特色历史地理功能：
- 对于史书中出现的历史地点，可视化该地点的具体信息。
- 对于史书中出现的历史人物，可视化该人物的人生轨迹。
- 对于史书中出现的历史政权，可视化该政权的势力范围演变。
# 未来展望
在特定的历史著作范畴之外，“史图记”团队还计划开发AI大语言模型，为任意一段用户上传的历史文本，提供可视化服务。

# "史图记·三国志" 地名标注指南

本文档面向所有希望为"史图记·三国志"项目做出贡献的志愿者，详细说明如何从《三国志》原文中识别地名、查证坐标，并按规范完成标注工作。

---

## 一、项目背景与目标

"史图记·三国志"是一个将《三国志》传记内容与地理信息系统相结合的可视化项目。项目的核心数据来源于两个文件：

| 文件 | 位置 | 作用 |
|------|------|------|
| `places.js` | `src/data/places.js` | 存储所有地名的经纬度和现代地名对照 |
| `传记文件.js` | `src/data/biographies/[魏\|蜀\|吴]/人物名.js` | 存储各人物传记原文，并在文中用 HTML 标签标记地名 |

**标注目标**：让《三国志》传记中每一个出现的历史地名，都能在地图上准确定位、可视化展示。

---

## 二、标注前的准备

### 2.1 领取任务

在开始之前，请通过项目协作渠道（如 GitHub Issues 或共享表格）领取一个**尚未完成标注**的传记文件，避免与他人重复劳动。

每次领取一篇传记，完成后再领取下一篇。

### 2.2 了解文件结构

**传记文件**（以刘备传为例）的基本格式：

```javascript
// src/data/biographies/shu/liubei.js
export default {
  id: 'shu-1-liubei',
  name: '刘备',
  content: {
    title: '刘备传',
    text: `
      <p>传记原文内容，地名已用 HTML 标签标记……</p>
    `
  }
}
```

**地点数据文件**（`places.js`）的基本格式：

```javascript
export const places = {
  '涿郡': { lat: 39.53, lng: 115.97, modernName: '河北省保定市涿州市' },
  // 更多地点……
}
```

## 三、标注流程（六步法）

### 第一步：通读原文，识别所有地名

打开你领取的传记文件，阅读 `text` 字段中的传记内容。

逐句阅读，用纸笔或文档标记所有**地名**。地名通常包括：

- **州**：冀州、益州、荆州、扬州……
- **郡**：南阳郡、涿郡、丹杨郡……
- **县/城**：许昌、下邳、成都、建业……
- **关隘/山川**：阳平关、祁山、赤壁、官渡……
- **具体地点**：某某营、某某坞、某某亭……

> **注意**：人名中含有地名（如"涿郡人"）也需要标注其地名部分。

---

### 第二步：逐一查证每个地名的现代坐标

对于每个识别出的地名，需要确定其对应的**现代地理位置**和**经纬度**。

#### 推荐查证方法

1. **优先查阅学术参考资料**：
   - 谭其骧主编《中国历史地图集》（三国时期分册）
   - 《中国地名大辞典》
   - 复旦大学历史地理研究所相关资料

2. **在线资源辅助核查**：
   - 中国历史地名（CHGIS）数据库
   - 维基百科（搜索词：地名 + 三国/东汉）
   - 百度百科词条核实

3. **获取经纬度**：
   - 推荐工具：[高德地图坐标拾取器](https://lbs.amap.com/tools/picker)
   - 坐标精度保留**小数点后两位**即可（例：`lat: 34.03, lng: 113.85`）

#### 坐标填写规范

| 字段 | 说明 | 示例 |
|------|------|------|
| `lat` | 纬度（北纬为正值） | `34.03` |
| `lng` | 经度（东经为正值） | `113.85` |
| `modernName` | 现代行政区划全称 | `'河南省许昌市'` |

> **精度说明**：坐标无需精确到村落级别，能定位到县市级即可。

---

### 第三步：在 `places.js` 中登记新地点

查证完坐标后，打开 `src/data/places.js`，检查该地名是否**已经存在**：

```javascript
// 用 Ctrl+F 搜索地名
'许昌': { lat: 34.03, lng: 113.85, modernName: '河南省许昌市' },
```

- **若已存在**：直接跳到第四步，无需重复添加。
- **若不存在**：在文件末尾（`}` 之前）新增一行：

```javascript
export const places = {
  // ……已有地点……
  '新地名': { lat: 纬度, lng: 经度, modernName: '省份+城市全称' },
}
```

**格式要求**：
- 地名使用**单引号**包裹
- `lat` 和 `lng` 为**纯数字**，不加引号
- `modernName` 使用**单引号**，填写省市区全称

---

### 第四步：在传记原文中用 HTML 标签标记地名

打开传记文件，在 `text` 字段中找到需要标注的地名，将其替换为以下格式：

```html
<a class="place-name [阵营]" data-place="地名" data-year="年份" data-faction="[阵营]">地名</a>
```

#### 各属性说明

| 属性 | 必填 | 说明 |
|------|------|------|
| `class` | 是 | `place-name` 固定不变，后接阵营标识 |
| `data-place` | 是 | 地名，**必须与 places.js 中的 key 完全一致** |
| `data-year` | 是 | 该事件发生的大致年份，格式为 `"XX年"` |
| `data-faction` | 是 | 该地当时所属的阵营（见下表） |

#### 阵营（faction）取值规则

| 值 | 含义 | 适用情境 |
|----|------|----------|
| `wei` | 魏国 | 该地处于曹魏控制下 |
| `shu` | 蜀国 | 该地处于蜀汉控制下 |
| `wu` | 吴国 | 该地处于东吴控制下 |
| `neutral` | 中立/未确定 | 东汉初期、三方拉锯地带、或无法判断阵营时 |

> **阵营判断原则**：以**事件发生时**（`data-year` 对应年份）该地的实际控制方为准，而非人物所属阵营。
>
> 例如：刘备（蜀）进入荆州（208年），当时荆州尚未明确归属，用 `neutral`。

#### 标注示例

**标注前（原文）**：

```javascript
text: `
  <p>先主走荆州，刘表收之。</p>
`
```

**标注后**：

```javascript
text: `
  <p>先主走<a class="place-name neutral" data-place="荆州" data-year="200年" data-faction="neutral">荆州</a>，刘表收之。</p>
`
```

---

### 第五步：自查与验证

完成标注后，进行以下自查：

#### 5.1 一致性检查

确认所有 `data-place` 的值都能在 `places.js` 中找到对应的 key：

```javascript
// places.js 中存在：
'荆州': { lat: 30.33, lng: 112.24, modernName: '湖北省荆州市' },

// 传记中标注正确（完全一致）：
data-place="荆州"  ✅

// 错误示例（拼写不一致）：
data-place="荆洲"  ❌
data-place="荊州"  ❌（繁体）
```

#### 5.2 格式检查

- `<a>` 标签是否完整闭合（有开始标签就必须有结束标签）
- 年份格式是否为 `"XXX年"`
- `class` 属性中的阵营值与 `data-faction` 是否**一致**

```html
<!-- 正确：class 和 data-faction 一致 -->
<a class="place-name wei" data-place="许昌" data-year="200年" data-faction="wei">许昌</a>

<!-- 错误：两者不一致 -->
<a class="place-name shu" data-place="许昌" data-year="200年" data-faction="wei">许昌</a>
```

#### 5.3 运行项目验证（可选）

如果你有本地开发环境，可以运行项目验证标注效果：

```bash
npm install   # 首次运行需要安装依赖
npm run dev   # 启动开发服务器，访问 http://localhost:3000
```

在浏览器中找到你标注的传记，点击标注的地名，确认地图能正确跳转到对应位置。

---

### 第六步：提交贡献

完成标注并自查无误后：

1. 确认修改了哪些文件（通常是 `places.js` 和对应的传记文件）
2. 通过 GitHub 提交 Pull Request，PR 标题格式为：

   ```
   标注：[书卷]-[人物名]传地名
   ```

   例如：`标注：蜀书-诸葛亮传地名`

## 四、特殊情况处理

### 4.1 地名无法考证

如果某地名经多方查证仍无法确定位置，请在提交时单独列出，标注为"存疑"。**不要猜测填写错误坐标**，可暂时跳过该地名。

### 4.2 同名异地

三国时期存在多处同名地点（如多处"临淮"、"武陵"）。需根据上下文（人物活动地域、前后叙述）判断指的是哪一处，在 `places.js` 中可用区分命名：

```javascript
'武陵(荆州)': { lat: 29.00, lng: 111.69, modernName: '湖南省常德市' },
'武陵(荆南)': { lat: 28.38, lng: 109.00, modernName: '湖南省湘西州' },
```

传记中的 `data-place` 也相应使用带区分的名称。

### 4.3 地名跨越多处或范围过大

对于"天下"、"中原"等泛指性词汇，**不需要标注**，只标注具体可定位的地名。

## 五、完整标注示例

以下是一段完整的标注示例，供参考：

**原文（《三国志·蜀书·诸葛亮传》节选）**：

> 亮躬耕陇亩，好为梁父吟。身长八尺，每自比於管仲、乐毅，时人莫之许也。惟博陵崔州平、颍川徐庶元直与亮友善，谓为信然。时先主屯新野。

**标注后**：

```javascript
text: `
  <p>亮躬耕<a class="place-name neutral" data-place="隆中" data-year="207年" data-faction="neutral">隆中</a>陇亩，好为梁父吟。身长八尺，每自比於管仲、乐毅，时人莫之许也。惟博陵崔州平、<a class="place-name neutral" data-place="颍川" data-year="207年" data-faction="neutral">颍川</a>徐庶元直与亮友善，谓为信然。时先主屯<a class="place-name neutral" data-place="新野" data-year="207年" data-faction="neutral">新野</a>。</p>
`
```

**places.js 中新增**：

```javascript
'隆中': { lat: 32.12, lng: 112.43, modernName: '湖北省襄阳市隆中风景区' },
'颍川': { lat: 34.05, lng: 113.22, modernName: '河南省许昌市禹州市一带' },
'新野': { lat: 32.52, lng: 112.36, modernName: '河南省南阳市新野县' },
```

## 六、常见错误汇总

| 错误类型 | 错误示例 | 正确做法 |
|----------|----------|----------|
| `data-place` 与 `places.js` key 不一致 | `data-place="荆洲"` 而 places.js 中是 `'荆州'` | 保证完全一致，包括繁简体 |
| 忘记在 places.js 中添加新地名 | 传记中有标注但 places.js 无对应条目 | 两步必须同时完成 |
| 阵营填写不符合实际 | 袁绍控制区域写成 `wei` | 以事件发生时的实际控制方为准 |
| 年份格式错误 | `data-year="200"` | 必须带"年"字：`data-year="200年"` |
| 标签未闭合 | `<a class="place-name wei" data-place="许昌">许昌` | 末尾必须有 `</a>` |
| 标注泛指词汇 | 将"中原"、"天下"添加标签 | 泛指词汇不标注 |

## 七、联系与协作

### 7.1 如何在 GitHub Issues 中认领任务

项目维护者会预先为每篇待标注传记创建一个 Issue（例如 `#12 待标注：魏书·张辽传`）。志愿者按以下步骤认领：

**第一步：找到可认领的 Issue**

打开项目 GitHub 页面，点击顶部的 **Issues** 标签，筛选带有 `help wanted` 标签且 Assignees 栏为空的条目。

**第二步：在 Issue 下评论认领**

确认没有其他人已在评论中声明认领后，发表评论：

```
我来认领这篇，预计 X 天内完成。
```

维护者看到后会将 Issue 指派（Assign）给你，其他人就不会重复认领同一篇。

**第三步：完成后提交 Pull Request**

在 PR 描述中写：

```
closes #Issue编号
```

例如 `closes #12`。PR 合并后该 Issue 自动关闭，任务完成。

### 7.2 如果还没有对应的 Issue

可以自己创建一个 Issue，标题格式为：

```
[标注认领] 魏书·张辽传
```

说明你将负责该传记，再开始工作。

### 7.3 疑难讨论

遇到无法判断的地名，在对应 Issue 下留言说明情况，维护者或其他志愿者会协助讨论。
